# SmoothScroll v0.3.0 — Polish + Auto-Update Spec

**Date:** 2026-05-16
**Status:** Approved — proceeding to writing-plans
**Target:** Windows .exe build

## 1. Goals

Four independent improvements:

1. **Scroll presets** — 6 quick presets on the Scroll tab so users don't have to tune every slider.
2. **Smaller window minimum** — let users resize the settings window down to 480×400.
3. **Auto-update** — version bumps automatically per push, app checks GitHub Releases, auto-downloads + installs new versions via Tauri's official updater.
4. **Test sandbox flex-fill** — the test scroll box fills available vertical space.

## 3. Component details

### 3.1 Scroll presets

Six presets that set step_size_px / animation_time_ms / acceleration_delta_ms / acceleration_max in one click. Active preset gets a highlighted border. Switching to a preset calls `patch({...})`.

| Preset | step_size | anim_time | accel_delta | accel_max | Notes |
|---|---|---|---|---|---|
| Slow | 80 | 600 | 100 | 4 | Calm, slow acceleration |
| Default | 120 | 360 | 70 | 7 | Current default |
| Fast | 160 | 280 | 50 | 10 | Quick, responsive |
| Snappy | 200 | 200 | 30 | 14 | Very fast, near-instant |
| Mac-like | 100 | 500 | 80 | 6 | Smooth + long curve |
| Linear | 120 | 360 | 0 | 1 | No acceleration, even pace |

Detection of "active" preset: deep-equal current settings against each preset; first match wins.

UI: row of 6 small chip-style buttons (`text-xs px-2 py-1 rounded-md border`) wrapped in a single row, scrolling horizontally if window narrow.

Position: at the top of `ScrollSection` card, above the first slider.

### 3.2 Window minimum size

`tauri.conf.json` → `minWidth: 480, minHeight: 400` on the `main` window (the settings window). Default size kept at 900×640. The recently-added `tray-panel` window (300×440, `resizable: false`) is unaffected.

After 25% font shrink + 176px sidebar, 480×400 leaves ~300×370 for content. Sliders are 192px wide → still fit. ScrollPresets row may wrap to 2-3 lines on narrow widths — fine.

### 3.3 Auto-update

**Tauri plugin-updater approach** (recommended by Tauri team).

Backend:
- Add `tauri-plugin-updater = "2"` to `src-tauri/Cargo.toml`.
- `src-tauri/src/lib.rs`: `.plugin(tauri_plugin_updater::Builder::new().build())`.
- `tauri.conf.json` plugins block:
  ```json
  "plugins": {
    "updater": {
      "active": true,
      "endpoints": [
        "https://github.com/quangtruong2003/SmoothScroll/releases/latest/download/latest.json"
      ],
      "dialog": false,
      "pubkey": "{{KEY}}"
    }
  }
  ```
- Pubkey: generate via `tauri signer generate -w ~/.tauri/myapp.key`. Public key embedded; private key stored in GitHub Actions secret `TAURI_SIGNING_PRIVATE_KEY`.
- Release workflow generates `latest.json` + `.sig` files alongside the .exe and includes them in the GitHub Release asset list.

Frontend `src/lib/updater.ts`:
```ts
import { check } from "@tauri-apps/plugin-updater";

export async function checkForUpdate() {
  const update = await check();
  return update; // { available, currentVersion, version, body, ... } or null
}

export async function installUpdate(update) {
  await update.downloadAndInstall((event) => {
    // event: { event: "Started" | "Progress" | "Finished", data }
  });
}
```

`AboutSection.tsx`:
- "Check for updates" button → calls `checkForUpdate()`.
- States: `idle | checking | up-to-date | available | downloading | done | error`.
- On `available`: shows `v{newVer}` + "Install now" button + release notes (from `update.body`, plain text).
- On `downloading`: progress bar (percent from `event.data.chunkLength / contentLength`).
- On `done`: button "Restart now" calls `relaunch()` from `@tauri-apps/plugin-process`.

Auto-check on app start: `useEffect` in `Settings.tsx` calls `checkForUpdate()` 5s after mount, silently (no toast). If available, surface a small badge on the About sidebar tab and on the AboutSection card. No popup.

### 3.5 Auto-tag CI

`.github/workflows/auto-tag.yml`:

```yaml
name: Auto Tag
on:
  push:
    branches: [master]
    paths:
      - 'crates/**'
      - 'src/**'
      - 'src-tauri/**'
      - 'package.json'
      - 'Cargo.toml'

permissions:
  contents: write

jobs:
  bump:
    if: "!startsWith(github.event.head_commit.message, 'chore: bump version')"
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
        with:
          fetch-depth: 0
          token: ${{ secrets.GITHUB_TOKEN }}

      - uses: dtolnay/rust-toolchain@stable

      - name: Install cargo-edit
        run: cargo install cargo-edit --locked

      - name: Bump workspace version
        id: bump
        run: |
          cargo set-version --bump patch -p smoothscroll_core -p smoothscroll_platform -p smoothscroll-app --workspace
          NEW_VERSION=$(grep -m1 '^version =' Cargo.toml | sed -E 's/.*"(.*)"/\1/')
          echo "version=$NEW_VERSION" >> $GITHUB_OUTPUT

      - name: Sync package.json
        run: |
          jq --arg v "${{ steps.bump.outputs.version }}" '.version = $v' package.json > package.json.tmp
          mv package.json.tmp package.json

      - name: Sync tauri.conf.json
        run: |
          jq --arg v "${{ steps.bump.outputs.version }}" '.version = $v' src-tauri/tauri.conf.json > src-tauri/tauri.conf.json.tmp
          mv src-tauri/tauri.conf.json.tmp src-tauri/tauri.conf.json

      - name: Commit + tag + push
        run: |
          git config user.name "github-actions[bot]"
          git config user.email "github-actions[bot]@users.noreply.github.com"
          git add -A
          git commit -m "chore: bump version to v${{ steps.bump.outputs.version }}"
          git tag "v${{ steps.bump.outputs.version }}"
          git push origin master
          git push origin "v${{ steps.bump.outputs.version }}"
```

Loop guard: `if: "!startsWith(github.event.head_commit.message, 'chore: bump version')"` prevents the bump-commit from re-triggering itself.

`release.yml` (existing) needs updates:
- Use latest tag's version in artifact filenames (currently uses `${{ github.ref_name }}` which is fine).
- Generate `latest.json` + `.sig` for updater feed.

`latest.json` schema:
```json
{
  "version": "0.1.5",
  "notes": "Release notes here",
  "pub_date": "2026-05-16T13:00:00Z",
  "platforms": {
    "windows-x86_64": {
      "signature": "...",
      "url": "https://github.com/quangtruong2003/SmoothScroll/releases/download/v0.1.5/SmoothScroll_0.1.5_x64-setup.exe"
    }
  }
}
```

Generated by Tauri build via `--config` flag or as a post-build step using the `.sig` files Tauri produces.

## 5. New IPC commands

None. All update logic uses Tauri's official `@tauri-apps/plugin-updater` JS API.

## 6. Migration / risk

- **Updater pubkey rotation:** if private key leaks, all clients fail to verify updates. Mitigation: store key in GH Secrets only; never commit.
- **Bump loop:** `chore: bump version` filter prevents recursion. If filter fails, GH Actions billing → kill switch via repo settings.
- **Min size + slider overflow:** test that 192px slider + label fits in 480-176-32 = 272px.
- **Test sandbox flex-fill:** if any settings card grows tall (e.g. ScrollPresets wraps), sandbox shrinks. Acceptable.
- **Auto-tag on every push:** every meaningful push bumps patch. Could noise the version space. Mitigation: `paths:` filter in workflow excludes docs-only changes.

## 7. Out of scope

- macOS auto-update (still stub backend)
- Code signing of .exe (separate task)
- Release notes auto-generation (uses existing `generate_release_notes: true`)
- Preset rename / custom presets / save-as-preset
