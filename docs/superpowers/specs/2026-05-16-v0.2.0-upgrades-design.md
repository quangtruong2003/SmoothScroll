# SmoothScroll v0.2.0 — Upgrade Spec

**Date:** 2026-05-16
**Status:** Draft, awaiting user review
**Target:** Windows .exe build first; macOS port deferred

## 1. Goals

Eliminate two known issues from v0.1.0 and ship four UX upgrades that materially improve daily use:

1. Custom global hotkey (replaces hard-coded `Ctrl+Alt+S`)
2. Live re-register hotkey when toggled in UI (no app restart)
3. Live tray menu localization when language changes
4. Tray icon reflects enabled/disabled state visually
5. Bidirectional Enable sync between tray, hotkey, and UI
6. Live easing curve preview (static graph)
7. Inline test sandbox in Settings
8. Light/Dark/System theme
9. Performance instrumentation for `is_excluded()`

## 3. Data flow

### 3.1 Hotkey lifecycle

```
[App start]
  load AppSettings
  if settings.enable_global_hotkey {
    register Accelerator(settings.hotkey_accelerator)
    store HotkeyHandle in AppState
  }

[User toggles "Global hotkey" switch]
  React → invoke set_hotkey_enabled(bool)
  Backend:
    if enabling   → register, store handle
    if disabling  → drop existing handle (RAII unregister)
  Settings persisted via debounced save_settings

[User changes hotkey_accelerator]
  React → invoke set_hotkey_accelerator(string)
  Backend:
    drop existing handle
    re-register with new accelerator
    save_settings

[User presses hotkey anywhere]
  Platform callback flips enabled
  emit "enabled-changed" event
  signal engine
```

### 3.2 Enable sync event

Single source of truth: `AppState.enabled: AtomicBool`. Every flip emits a Tauri event.

```
[UI Switch flip] → set_enabled(b) → emit "enabled-changed"
[Tray click]      → state.enabled.store(b) → emit "enabled-changed"
[Hotkey press]    → state.enabled.store(b) → emit "enabled-changed"

Frontend subscribers:
  - settingsStore listens, updates `settings.enabled` (no save)
  - tray icon listener (backend-side) swaps icon
```

### 3.3 Tray live i18n

```
[User changes language]
  React → invoke change_language(lang)
  Backend:
    save settings
    rebuild_tray_menu(lang)
    set_menu(new_menu)
  React i18next.changeLanguage(lang) → UI re-renders
```

## 4. Component details

### 4.1 HotkeyRecorderInput.tsx

Press-to-record input, similar to VS Code's keybindings UI.

States:
- `idle`: shows current accelerator like `Ctrl+Alt+S`. Click → enters `recording`.
- `recording`: input shows "Press a key combination…" + cancel button. Captures `keydown` globally on the input.
  - Tracks pressed modifiers (Ctrl / Alt / Shift / Win) + the most recent non-modifier key.
  - On `keyup` of any non-modifier with at least one modifier present → commit.
  - Esc → cancel back to `idle`.
- Validation: `is_valid_accelerator()` Rust-side; UI shows error if invalid (but validator runs on commit, not per keystroke).

Output format: `Modifier+Modifier+Key` (e.g. `Ctrl+Alt+S`, `Ctrl+Shift+F12`). Modifier order is normalized to `Ctrl, Alt, Shift, Win`.

### 4.2 EasingCurvePreview.tsx

Static SVG (more crisp than canvas, theme-friendly via `currentColor`).

```
viewBox: 0 0 240 80
X axis: t ∈ [0, 1]   (animation time)
Y axis: 1 - (1 - frac(t))   (cumulative emit)
```

Computes 60 sample points using same `compute_easing_fraction` formula in TS. Watches `easing_mode` and `tail_to_head_ratio`. Re-renders only when those values change (memo).

Shown in `AppearanceSection` directly under the easing-curve dropdown.

### 4.4 Theme system

`lib/theme.ts`:
```typescript
export type ThemeMode = "Light" | "Dark" | "System";

export function applyTheme(mode: ThemeMode) {
  const resolved = mode === "System"
    ? (matchMedia("(prefers-color-scheme: dark)").matches ? "Dark" : "Light")
    : mode;
  document.documentElement.classList.toggle("dark", resolved === "Dark");
}

export function watchSystemTheme(onChange: () => void) {
  const mq = matchMedia("(prefers-color-scheme: dark)");
  mq.addEventListener("change", onChange);
  return () => mq.removeEventListener("change", onChange);
}
```

`ThemeSection` lives at the top of the Appearance card (or its own card). Three radios: Light / Dark / System.

Tailwind config already supports `dark:` classes in shadcn theme.

### 4.6 Performance instrumentation

Trong `hook_wiring.rs::is_excluded()`:
```rust
fn is_excluded(&self) -> bool {
    let start = std::time::Instant::now();
    let result = ...;
    let elapsed = start.elapsed();
    if elapsed > Duration::from_millis(2) {
        tracing::debug!(?elapsed, "is_excluded slow");
    }
    result
}
```

Plus: skip lookup entirely when `excluded_apps` is empty (common case, free perf win):
```rust
fn is_excluded(&self) -> bool {
    if self.state.settings.read().excluded_apps.is_empty() {
        return false;
    }
    ...
}
```

Both apply. Dead code `snapshot_settings` deleted.

## 6. New IPC commands

| Command | Args | Returns | Purpose |
|---------|------|---------|---------|
| `set_hotkey_accelerator` | `accel: String` | `Result<(), String>` | Validate, save, re-register hotkey |
| `set_hotkey_enabled` | `enabled: bool` | `Result<(), String>` | Toggle hotkey registration |
| (existing) `set_enabled` | `enabled: bool` | `()` | Now also emits `enabled-changed` |
| (existing) `change_language` | `lang: String` | `Result<(), String>` | Now also rebuilds tray menu |

## 7. New Tauri events

| Event name | Payload | Emitter |
|------------|---------|---------|
| `enabled-changed` | `bool` | tray, hotkey, set_enabled command |

Frontend subscribes once in `App.tsx` (or in `settingsStore.load()`) and updates store.

## 8. Testing strategy

| Layer | Test | Tool |
|-------|------|------|
| Core | New `is_valid_accelerator()` cases | Rust `#[test]` |
| Core | `clamp()` resets bad accelerator | Rust `#[test]` |
| Core | `clamp()` resets bad theme | Rust `#[test]` |
| Hook wiring | empty excluded_apps short-circuits | Rust `#[test]` (existing test pattern in hook_wiring.rs) |
| Frontend | Manual smoke: each upgrade clickable + works | Run dev build |
| Build | `cargo tauri build` produces NSIS .exe | Local Windows machine |

No new automated frontend tests — existing project has none, adding a Vitest setup is out-of-scope per "surgical changes".

## 9. Migration / risk

- **JSON forward compat:** new fields default-fill from `serde(default)` on `AppSettings`. Old settings files load fine.
- **Hotkey conflict:** if user picks an accelerator already taken by another app, `RegisterHotKey` returns failure. Backend surfaces error string; frontend shows toast and reverts to previous accel.
- **Tray icon regen:** if icon generation fails at compile time, fall back to using same icon for both states (no visual feedback but no crash).
- **Theme + Tauri webview:** Tauri 2 webview already follows `prefers-color-scheme` for system; we just add user override.
