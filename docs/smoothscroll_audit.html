01

Executive Summary

SmoothScroll là landing page giới thiệu một tiện ích Windows giúp cá nhân hóa hành vi cuộn chuột (scroll wheel) trên mọi ứng dụng. Website được xây dựng bằng Next.js 14 (App Router) kết hợp Tailwind CSS, host tĩnh qua GitHub Pages. Trang đơn (single-page) gồm 12 section chính: hero, problem statement, feature grid, system tray showcase, indie dev bio, install steps, FAQ accordion, và footer CTA. Hỗ trợ đa ngôn ngữ (EN/VI/ZH) và dark/light mode.

Nhìn chung, đây là một sản phẩm chất lượng cao ở tầng visual design — typography rõ ràng, spacing thoáng, gradient & dark mode đẹp mắt, CTA hierarchy tốt. Tuy nhiên, vẫn còn một số vấn đề về accessibility (32 icon thiếu alt text), responsive (h1 96px gây horizontal scroll trên mobile 390px), và performance (tổng transfer ~30 MB do load 22 icon SVG từ iconify.design CDN mỗi lần tải trang).

UX Score — 78/100

Điểm UX được tính dựa trên 5 tiêu chí: clarity of value proposition (15/20), information hierarchy (14/20), task completion flow (15/20), error prevention & feedback (12/20), và learnability (12/20). Trang ghi điểm cao ở phần value proposition — tiêu đề H1 "Mouse wheel scrolling, finally done right." ngắn gọn, đánh trúng nỗi đau của người dùng Windows. Hero section có hai CTA rõ ràng: primary "Download for Windows" và secondary "See how it works", đúng best practice về visual hierarchy.

Điểm bị trừ chủ yếu ở error prevention & feedback và learnability. Trang không có form nào nên không có inline validation để đánh giá, nhưng tab "Linux" và "macOS" trong phần install steps bị disabled mà không có giải thích — người dùng click vào sẽ không có phản hồi gì, gây bối rối. Nút copy code (PowerShell installer) khi click chỉ đổi icon tạm thời chứ không có toast/aria-live thông báo "Copied!", vi phạm WCAG 4.1.3 Status Messages. Ngoài ra FAQ accordion hoạt động đúng nhưng không cho phép mở nhiều mục cùng lúc (single-expand), hạn chế khả năng so sánh câu trả lời.

Điểm mạnh

  • Hero CTA hierarchy rõ ràng — primary button đậm màu, secondary button text link không viền
  • Install steps numbered 1-2-3 với iconography nhất quán, code block có nút Copy
  • FAQ đầy đủ 12 câu hỏi bao phủ mọi thắc mắc của user (anti-cheat, UWP, uninstall...)
  • Brand marquee hiển thị 16 app tương thích — tăng trust signal

Điểm yếu

  • Tab Linux/macOS disabled không có tooltip giải thích — vi phạm heuristic Nielsen #10 (Help)
  • Nút Copy không có aria-live region thông báo trạng thái "Copied" cho screen reader
  • FAQ accordion single-expand — giảm khả năng compare
  • Không có scroll-to-top button khi user cuộn xuống FAQ (page dài ~6700px)
  • Section "Made by an indie dev" thiếu social proof (số GitHub star, download count)

UI Score — 82/100

Điểm UI đánh giá 4 yếu tố: visual design & polish (22/25), typography system (20/25), color & contrast (18/25), consistency & spacing (22/25). SmoothScroll ghi điểm rất cao về visual polish — gradient xanh-tím nhẹ ở hero, brand marquee với mask fade hai bên, trước/sau khi cài đặt dùng GIF minh họa, system tray screenshot chân thực. Responsive — Desktop / Tablet / Mobile

Test trên 3 viewport: Desktop 1440×900, Tablet 768×1024 (iPad portrait), Mobile 390×844 (iPhone 14). Kết quả: trang responsive TỐT trên desktop và tablet, nhưng có 2 lỗi overflow trên mobile cần sửa gấp. Container max-width 1400px hoạt động đúng trên desktop. Tailwind grid breakpoints (sm:640, lg:1024) chuyển đổi mượt mà, brand marquee scale phù hợp.

Phân tích root cause

  • Brand marquee scrollWidth 3990px trong container 294px — đã có overflow:hidden + mask nên OK về visual nhưng vẫn report
  • Container padding px-4 (16px) đủ cho mobile, không cần giảm thêm

Navigation — Header / Menu

Header sticky ở top, height 68px, border-bottom nhẹ. Cấu trúc: [Logo SmoothScroll] — [GitHub button] — [Language switcher EN] — [Theme toggle]. Không có menu chính (no main nav links) — đây là lựa chọn design hợp lý cho single-page landing với chỉ 1 CTA chính (Download).

  • Logo SmoothScroll là link về #top — OK, có aria-label thông qua img alt
  • GitHub button ở header và GitHub link ở footer cùng text "GitHub" nhưng khác href — không có aria-label phân biệt
  • Language switcher: button có aria-haspopup='menu', aria-label rõ — nhưng menu dropdown không có role='menu'/'menuitem'. Vi phạm WAI-ARIA Menu Pattern
  • Theme toggle: aria-label='Toggle theme' — OK nhưng không có aria-pressed cho biết trạng thái hiện tại
  • Mobile: header vẫn giữ layout ngang, không có hamburger menu — vì không có main nav nên OK
  • Sticky header: khi scroll, header vẫn visible — tốt cho navigation

Animations & Motion

Animation trên trang dùng chủ yếu CSS transition + Tailwind utility (transition-colors, transition-transform duration-200). Phát hiện: hover state trên links/buttons đổi màu mượt (200ms ease), FAQ accordion expand/collapse dùng CSS height transition, brand marquee scroll dùng CSS animation infinite. Không phát hiện animation nặng như parallax scroll hay WebGL — tốt cho performance.

  • Hover transitions: 200ms ease, đổi màu text-muted → text-foreground — phù hợp
  • FAQ accordion: height transition với overflow hidden — smooth, OK
  • Brand marquee: CSS keyframe animation infinite scroll — không có pause on hover (nên có)
  • Theme toggle: đổi màu ngay không fade — nên thêm transition 300ms
  • Page load: không thấy skeleton/loading state — Next.js SSG nên OK
  • Respect prefers-reduced-motion: CHƯA kiểm tra — cần verify

Đề xuất — tuân thủ WCAG 2.3.3 Animation from Interactions

@media (prefers-reduced-motion: reduce) {
  *, *::before, *::after {
    animation-duration: 0.01ms !important;
    transition-duration: 0.01ms !important;
  }
}

Đề xuất sửa — Mockup Before / After

Phần này trình bày 5 đề xuất sửa lỗi ưu tiên cao nhất, kèm giải thích lý do và cách implement.

17.1 — Sửa icon thiếu alt text (Bug #1)

✕ BEFORE — icon không có alt
Win · macOS · Chrome · VS Code
alt="" alt="" alt="" alt=""
Screen reader bỏ qua hoàn toàn hoặc chỉ đọc "image".
✓ AFTER — alt mô tả rõ
alt="Windows 11" · alt="macOS" · alt="Google Chrome" · alt="Visual Studio Code"
Screen reader đọc rõ tên app → đạt WCAG 1.1.1.

Giải thích: 32/36 icon brand đều có alt="" (empty) — người dùng khiếm thị không biết app nào đang được liệt kê. Implement trong component IconBrand: alt={name}. Effort: 30 phút.

17.2 — Sửa H1 quá lớn trên mobile (Bug #3)

✕ BEFORE — h1: 96px trên mobile 390px
Mouse wheel scrolling, finally done right.
Text tràn viewport → horizontal scroll 13px.
✓ AFTER — h1: clamp(36px, 8vw, 64px)
Mouse wheel scrolling, finally done right.
Responsive clamp → không overflow, vẫn giữ hierarchy.

Giải thích: đổi sang clamp(36px, 8vw, 64px) — responsive mượt mà, vẫn giữ visual impact trên desktop. Letter-spacing giảm từ -1.92px thành -0.8px cho mobile. Effort: 5 phút.

17.3 — Sửa focus outline mờ (Bug #8)

✕ BEFORE — focus không rõ ràng
Tab vào nút → không có outline rõ.
✓ AFTER — outline 3px solid blue + offset 2px
Đạt WCAG 2.4.7 Focus Visible. Contrast 4.5:1 với mọi background.

Giải thích: thêm CSS :focus-visible { outline: 3px solid #2563EB; outline-offset: 2px; } cho mọi interactive element. Effort: 10 phút.

17.4 — Sửa code block overflow mobile (Bug #3 phụ)

✕ BEFORE — overflow-x 412px trong container 234px
irm https://github.com/quangtruong2003/Smo...
User phải scroll ngang — khó đọc trên mobile.
✓ AFTER — wrap URL + giảm font + nút Copy nổi bật
irm https://github.com/quangtruong2003/SmoothScroll/releases/latest/download/install.ps1 | iex
white-space: pre-wrap + word-break: break-all.

Giải thích: đổi white-space: pre sang pre-wrap + word-break: break-all cho URL dài. Kết hợp giảm font size 14px → 11px và nút Copy nổi bật hơn. Effort: 15 phút.

17.5 — Sửa dark mode background (Bug #5)

✕ BEFORE — dark mode nhưng body bg = white
Feature
Per-monitor profiles
Dark mode active nhưng html background vẫn rgb(255,255,255) → white flash.
✓ AFTER — html/body bg = #09090b khi dark
Feature
Per-monitor profiles
Background đồng nhất tối — loại bỏ white flash.

Giải thích: khi theme=dark, html background vẫn là rgb(255,255,255). Đề xuất: set html, body { background: #09090b } khi data-theme=dark. Effort: 5 phút.
